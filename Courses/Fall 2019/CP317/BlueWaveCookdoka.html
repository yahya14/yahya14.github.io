    2.1 The Purpose:

    1. This is a web-based application with two main components. First, the frontend is responsible for getting user’s information to log them in or to get a list of their ingredients and finally showing them a recipe. The other being the backend that has database of recipes, which gets sent back to the front upon finding a recipe match based on the user’s ingredient list, and user profiles which allows the users to give their feedback on a recipe. .

    2.2 Product functions:

    1. Search a recipe based on ingredients available
    2. Search recipe relating to a diet
    3. Creating a profile and saving/pinning user’s favourite recipe
    4. Rating and commenting on a recipe .

    • 3.1 diagram:
    User System
    Rating System Functional To review anything they have to be signed in.
      • validate the review (turn it into plain text).
      • Check the length of the review.
      • Check that the review does not have any code in it.
      • Check for profanity in the review.
      • Check for profanity in the review.
    Non functional Must be a certain length (100 characters)
      • Logs users out after a certain time of inactivity
      • Accept one request even if clicked to submit multiple times within a time frame (20 seconds)
    Login Functional Will sign user into website.
      • Enter credentials
      • Query database with credentials
      • Return user login status to page and initiate session
    Non functional Password and username must have certain length(32 max)
      • Must query database in short amount of time (<3000 MS)
    logout Functional
      • Logs user out when they click logout.
      • User must be logged in.
      • Invalidate session.
    Non functional • If they already timed out they cannot logout • After 60 minutes invalidate session
    add recipes Functional
      • User clicks on add recipes.
      • Enters recipe information then submit.
      • Give user form to enter recipe information(name, category, ingredients, instructions, image url)
    Non functional • User must be logged in.
      • Check if user is logged in
      • Check the size of all the categories in the form to make sure they do not enter too much data.
      • Insert into database
    Search Functional
      • User fills in search criteria and click search.
      • The search criteria is passed to the backend.
      • The criteria is used to query the database.
      • The result is returned to the frontend to be displayed.
    Non functional • Time out if the search takes too long (>10000MS).
